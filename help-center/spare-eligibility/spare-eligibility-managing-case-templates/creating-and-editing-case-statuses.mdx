Spare's color-coded case statuses allow you to easily monitor and manage the progress of your cases through a visual workflow system.

<Info>
You can create or edit case statuses within your Case Template. Navigate to the **Case Templates** tab, select the template you'd like to work on, then scroll to the **"Case Statuses"** section at the bottom of the page.
</Info>

## Default Case Statuses

By default, every case template includes three standard statuses:

<CardGroup cols={3}>
  <Card title="To Do" icon="circle" iconType="regular">
    **Grey** - Initial status for new cases
  </Card>
  <Card title="In Progress" icon="clock" iconType="regular">
    **Brown** - Active cases being worked on
  </Card>
  <Card title="Done" icon="check-circle" iconType="solid">
    **Green** - Completed cases
  </Card>
</CardGroup>

## Managing Case Statuses

### Editing Existing Statuses

You can customize both the name and color of any case status:

<Steps>
  <Step title="Access Status Settings">
    Click the **"Edit"** button next to the status you want to modify.
  </Step>
  
  <Step title="Update Properties">
    Modify the status name and select a new color from the color picker.
  </Step>
  
  <Step title="Save Changes">
    Click **"Save"** to apply your changes to the status.
  </Step>
</Steps>

<img src="/images/spare-eligibility/managing-case-templates/edit-add-status-buttons.png" alt="Case template screen with Edit and Add Case Status buttons highlighted" />

### Adding New Statuses

<Steps>
  <Step title="Add Status">
    Click **"+ Add Case Status"** to create a new status.
  </Step>
  
  <Step title="Configure Status">
    Enter the status name and select an appropriate color that represents the workflow stage.
  </Step>
  
  <Step title="Save Template">
    Remember to save your template changes after adding new statuses.
  </Step>
</Steps>

<Tip>
Choose colors that create a logical visual progression through your workflow. Consider using cooler colors (blue, purple) for early stages and warmer colors (orange, red) for urgent or final stages.
</Tip>

### Reordering Statuses

You can customize the order of your case statuses to match your workflow:

<Note>
**Reordering Limitations**: All statuses except the first and last ones can be reordered. The first status serves as the default for new cases, while the last status typically represents completion.
</Note>

To reorder statuses:
1. **Drag and drop** statuses to your desired position
2. **Save changes** to preserve the new order

## Best Practices

<Accordion title="Status Design Guidelines">
  <AccordionItem title="Naming Conventions">
    - Use clear, action-oriented names (e.g., "Under Review", "Awaiting Documentation")
    - Keep names concise but descriptive
    - Maintain consistency across all case templates
  </AccordionItem>
  
  <AccordionItem title="Color Strategy">
    - Use intuitive color associations (red for urgent, green for complete)
    - Ensure sufficient contrast for accessibility
    - Avoid using too many similar colors that could cause confusion
  </AccordionItem>
  
  <AccordionItem title="Workflow Design">
    - Design statuses to reflect your actual business process
    - Include all necessary intermediate steps
    - Consider adding statuses for common bottlenecks or decision points
  </AccordionItem>
</Accordion>
