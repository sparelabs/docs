### üìã Case Type Templates for Quick Setup

**What's New**  
Backend infrastructure is now in place for Case Type templates, enabling the rapid creation of pre-configured case types. The first available template is for "Incident" reporting, which automatically sets up the case type, statuses, a custom form, and relevant object links.

**Pre-requisites**
- Requires access to the Spare Resolve product
- Requires Admin access with permissions for Engage/Resolve

<Accordion title="See More">

#### Where You'll See It
- This is currently a backend and API-level feature. Activation is performed via API, with a user interface to follow in a future release.

#### What You Can Do
- **Activate Templates via API:** Use the new `/engage/caseTypeTemplates` endpoints to list available templates and activate them.
- **Auto-Create "Incident" Case Type:** Activating the `incident` template automatically creates:
  - An "Incident" case type.
  - A linked form with fields for `Incident Datetime`, `Description`, `Severity`, and `Category`.
  - Object links to associate incidents with a `Duty` or `Request`.
- **Intelligent Re-activation:** If you activate a template for a case type that was previously created and then archived, the system will unarchive the existing case type instead of creating a duplicate.

#### Why This Matters
Setting up a new case type with all its associated forms and fields can be time-consuming. Templates streamline this process, allowing administrators to deploy standardized configurations like incident management with a single action, ensuring consistency and saving time.
</Accordion>

### üîç Case Intelligence Summary

**What's New**  
This release introduces an AI-powered summary panel on the Case view page. This panel automatically generates a concise overview of the case from its description and tags, suggests relevant actions, and provides insights into the case details.

**Pre-requisites**
- Requires `CaseIntelligence` feature flag
- Requires Admin access

<Accordion title="See More">

#### Where You'll See It
- *Engage > Cases > [Select a Case]* (new "Case Intelligence" panel on the right side)

#### What You Can Do
- Click the "Generate summary" button to create an AI-generated summary of the case.
- View suggested tags, priority changes, and potential duplicate cases.
- Apply suggested actions with a single click.
- Use the "Thinking" peek to see the AI's step-by-step reasoning for its suggestions.

#### Why This Matters
Manually reviewing long case histories is time-consuming. This feature uses AI to quickly synthesize case information, identify key details, and suggest next steps, helping agents resolve cases faster and more consistently.
</Accordion>

### ‚òëÔ∏è Multi-Select for Priority and Tags in Triage Rules

**What's New**  
When building Triage Rules, you can now select multiple values for "Priority" and "Tags" conditions. The interface now uses a multi-select dropdown with checkboxes for these fields when using operators like "is one of" or "is none of."

**Pre-requisites**
- Requires Admin access
- No feature flags required

<Accordion title="See More">

#### Where You'll See It
- *Engage > Triage > [Case Type] > Add Rule / Edit Rule*: The value input for **Priority** and **Tags** conditions.

#### What You Can Do
- **Create multi-value conditions:** For example, you can create a rule where "Priority is one of [Urgent, High]".
- **Use new operators:** The rule builder now supports multi-value operators like `is one of`, `is none of`, and `is all of`.
- **Easily select multiple items:** A new dropdown component with a checkbox list allows you to select or deselect multiple priorities or tags. The trigger button will show a count of selected items (e.g., "2 priorities selected").

#### Why This Matters
This change makes Triage Rules more powerful and flexible. Instead of creating separate rules for each priority or tag, you can now group them into a single, more efficient rule, simplifying your triage automation setup.
</Accordion>

### ‚è±Ô∏è Auto-Refreshing Case Lists

**What's New**  
The case list pages in both Eligibility and Resolve now automatically refresh in the background. This ensures that you always see the most up-to-date case information, such as new assignments from triage rules, without needing to manually refresh the page.

**Pre-requisites**
- No feature flags required

<Accordion title="See More">

#### Where You'll See It
- *Eligibility > Cases > All Cases*
- *Resolve > Cases > All Cases*

#### What You Can Do
- Previously, if a case was assigned to you or its status changed via an automated triage rule, you had to refresh the page to see the update.
- Now, the case list and the summary counts (e.g., 'Assigned to Me', 'To Do') will update automatically every 60 seconds while the page is open and visible.

#### Why This Matters
This eliminates the need for manual page refreshes and reduces the chance of agents working on outdated information. It creates a more dynamic and real-time experience, ensuring case assignments and status changes are reflected almost immediately.
</Accordion>

### ‚öñÔ∏è Configure Case Triage and Assignment Rules

**What's New**  
A new "Triage" configuration page allows you to define automated assignment rules for incoming cases based on their properties, streamlining your case management workflow.

**Pre-requisites**
- Requires Admin access. Editing rules requires the `Create, update, or delete any Triage Definition` (`WRITE_ANY_TRIAGE_DEFINITION`) permission.

<Accordion title="See More">

#### Where You'll See It
- *Engage > Settings > Triage*

#### What You Can Do
- For each Case Type, configure a set of rules that determine how a new case is assigned.
- Set a default assignee (a user, user group, or queue) and an assignment method (Round Robin or Load Balancing).
- Use the Rule Builder to create conditional logic. For example, "IF `Tags` contains `Complaint` AND `Priority` is `High`, THEN assign to `Escalations Queue`."
- For the Eligibility product, the rule builder is simplified to only configure the trigger, default assignee, and assignment method.

#### Why This Matters
This automates the process of routing cases to the correct team or individual, reducing manual effort and ensuring cases are handled promptly by the right people.
</Accordion>

### üìù Include Trip History in Case Letter Templates

**What's New**  
You can now include a rider's trip history directly in case letter templates using new variables. These variables allow you to create lists of trips filtered by status (e.g., `completed`, `noShow`) and time (e.g., `lastWeek`, `lastMonth`), perfect for generating invoices or trip summaries.

**Pre-requisites**
- Requires Admin access with permissions for the Resolve (Engage) module.

<Accordion title="See More">

#### Where You'll See It
- *Resolve > Letters > Templates > [New/Edit Template]*: The variable picker now includes new `requests` variables.

#### What You Can Do
- Build templates that automatically pull in a rider's completed trips from the last month.
- Generate letters that list all no-show incidents for a specific rider.
- Use Handlebars logic (e.g., `{{#each requests.lastMonth.completed}}`) to format trip lists into tables or summaries.

#### Why This Matters
Manually compiling trip history for invoices or communications is time-consuming. This feature automates the process by allowing you to embed dynamic, filtered trip lists directly into your letter templates, saving time and reducing errors.
</Accordion>

<Update label="18 November 2025" description="1 Updates">

### ‚úÖ Case Management Features Now Standard

**What's New**  
Several features within Resolve (the new name for Engage) are now standard for all users and no longer require a feature flag. This includes the Activity Feed, the "Peek" view for forms and letters, and Conditional Form logic.

**Pre-requisites**
- Requires Admin access
- No feature flags required (previously required `EngageActivityFeed`, `EngageCaseViewV2Peek`, and `ConditionalForms` feature flags)

<Accordion title="See More">

#### Where You'll See It
- *Resolve > Cases > [Select a Case]* (the **Activity Feed** is now the default view instead of the "Activity Log" tab).
- *Resolve > Forms > [Form Builder]* (conditional logic options are always available).

#### What You Can Do
- The modern **Activity Feed** is now the standard interface for viewing case history.
- The ability to quickly **"Peek"** at forms and letters without leaving the main view is enabled by default.
- **Conditional logic** in the Form Builder is now available to all, allowing you to create dynamic forms that show or hide fields based on user input.

#### Why This Matters
By making these mature features standard, we are providing a more consistent and powerful user experience for everyone using Resolve to manage cases.
</Accordion>
