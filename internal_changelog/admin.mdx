### ‚öôÔ∏è Compare Dates in Workflows with New Helpers

**What's New**  
Two new Handlebars helpers, `parseDate` and `parseDatetime`, are now available in Workflows. These allow you to convert date and datetime strings from custom fields or other data sources into a numerical format (Unix timestamp) for comparisons in Filter steps.

**Pre-requisites**
- No feature flags required
- Requires Admin access

<Accordion title="See More">

#### Where You'll See It
- *Settings > System > Workflows* (within the "Filter" step of a workflow configuration)

#### What You Can Do
- Use `{{parseDate 'YYYY-MM-DD'}}` to convert a date string into a timestamp, correctly handled in your organization's timezone.
- Use `{{parseDatetime 'ISO_8601_STRING'}}` to convert a full datetime string into a timestamp.
- **Example Scenario**: Create a workflow that sends an email 30 days before a driver's license expires. In a Filter step, you can now compare a `driversLicenseExpiry` custom field to the current date by checking if `{{parseDate driver.metadata.driversLicenseExpiry}}` is less than `{{addTs (currentTs) 30 'days'}}`.

#### Why This Matters
Previously, you could not perform date-based comparisons in workflow filters because they only supported numerical values. This enhancement unlocks new automation possibilities, such as triggering actions based on upcoming expiry dates stored in custom fields.
</Accordion>
