### üë• Unarchive Group Memberships

**What's New**  
Admins can now unarchive a rider's archived group membership. This restores the membership, making it active again, provided it doesn't conflict with another existing active membership for the same group.

**Pre-requisites**
- Requires Admin access
- No feature flags required

<Accordion title="See More">

#### Where You'll See It
- *System > Data > Riders > [Select Rider] > Group Memberships tab* (a new **Unarchive** button appears in the actions menu for archived memberships).
- *System > Data > Group Memberships > [Select Archived Membership]* (a new **Unarchive** button appears at the top of the page).

#### What You Can Do
- Click the "Unarchive" button on an archived group membership.
- Confirm the action in the popup dialog.
- The membership will be restored to an active state. If this would create an overlap with an existing active membership for the same group, the action will be blocked and an error message will be displayed.

#### Why This Matters
Previously, if a group membership was archived by mistake, there was no easy way to restore it. This feature provides a simple, one-click solution for admins to correct errors and reactivate memberships, improving administrative efficiency.
</Accordion>
