### ‚ôø New Accessibility Features for Uber Fleets

**What's New**  
When configuring an Open Fleet with Uber as the provider, you can now specify additional accessibility options: `ActualDoorToDoor`, `ChildSeatBaby`, and `ChildSeatYoung`.

**Pre-requisites**
- Requires Admin access to manage Fleets

<Accordion title="See More">

#### Where You'll See It
- *Spare Operations > Service Planning > Fleets* > Add or Edit an Open Fleet with provider "Uber".

#### What You Can Do
- In the fleet settings, enable the **"Accessibility Features"** checkbox.
- Click **"Add Accessibility Features"**.
- The new options will appear in the **"Type"** dropdown menu, alongside existing options like `Wheelchair`.

#### Why This Matters
This enhancement allows you to offer more granular accessibility services for trips fulfilled by Uber. By passing these specific requirements, you can ensure riders are matched with vehicles that meet their exact needs, such as those that can accommodate young children or provide door-to-door assistance.
</Accordion>

### üîç Search for User Groups in Global Search

**What's New**  
The global search bar (Superbox) at the top of the Admin portal can now find and display Authorization Groups (user groups). Search results for groups will show the group name and description, and clicking a result takes you directly to its configuration page.

**Pre-requisites**
- Requires `RBAC` feature flag
- Requires Admin access

<Accordion title="See More">

#### Where You'll See It
- The global search bar at the top of any page in Spare Admin.

#### What You Can Do
- Type the name of a user group into the search bar.
- View matching groups in the search results, identified with a group icon.
- Click a group in the results to navigate directly to its details page at `/admin/authorization-groups/:id`.

#### Why This Matters
This makes it faster and easier for administrators to find and manage user groups without having to navigate through the *Settings* menu.
</Accordion>

### üîë Just-in-Time (JIT) Access Requests for Organizations

**What's New**  
A new self-serve system allows Spare employees (Super Admins) to request temporary, role-based access to customer organizations. This workflow includes justification, approvals, and automatic revocation, enhancing security and auditability.

**Pre-requisites**
- Requires Super admin permissions

<Accordion title="See More">

#### Where You'll See It
- The organization selection page after login (*Select Organization*)
- *Super Admin > Organizations > (select an org) > 'Request Access' button*
- A new top-level *Access Requests* page in the sidebar

#### What You Can Do
- Request access to an organization for a specific duration and role, with a justification.
- Designated approvers are notified to approve or deny the request.
- Track the status of your requests and see when access will expire.
- Manage all access requests and approval policies from the new "Access Requests" page.

#### Why This Matters
This replaces manual access grants with a structured, auditable, and time-bound system. It ensures that access to customer environments is granted on a need-to-know basis with the principle of least privilege, improving security and compliance.
</Accordion>

**Pre-requisites**
- Requires Admin access

<Accordion title="See More">

#### Where You'll See It
- *Settings > Payment > Payment Providers > (creating or editing a Stripe provider)*

#### What You Can Do
- When you enter Stripe API keys, the system will prevent you from saving if the keys are invalid or if there's a mismatch between Live and Test environments, showing a specific error message.

#### Why This Matters
This prevents misconfigurations that could lead to payment processing failures. By catching errors during setup, it ensures that the payment integration is correctly configured before it's used.
