### üìù Improved Form and Custom Field Configuration

**What's New**  
We've introduced several quality-of-life improvements for managing custom fields and forms. You can now reorder choice options via drag-and-drop, edit the labels of existing choices without changing their underlying value, and quickly identify required fields in your forms with a new "Required" indicator.

**Pre-requisites**
- Requires Admin access

<Accordion title="See More">

#### Where You'll See It
- **Choice Reordering & Editing**: *Settings > Field Manager > [Select a Single/Multiple Choice Field]* (in the "Choices" section). This also applies to choice-based fields within the Engage form builder at *Engage > Case Types > [Select Case Type] > Forms*.

#### What You Can Do
- **Reorder Choices**: When configuring a "Single Choice" or "Multiple Choice" custom field, use the new drag handle (grip-lines icon) to the left of each option to change its display order.
- **Edit Choice Labels**: Click the new pencil icon next to a choice option to edit its public-facing label. The underlying value/key remains read-only to ensure data integrity.
- **Identify Required Fields**: When viewing fields in the form builder, a "Required" indicator now appears next to the visibility text (e.g., "Editable and visible to Riders ¬∑ Required"), making it easy to see which fields are mandatory at a glance.

#### Why This Matters
Previously, managing choice-based custom fields was rigid; reordering options required deleting and re-creating them, and labels couldn't be edited. These updates make form and custom field management more flexible and intuitive, saving administrators significant time and effort.
</Accordion>

This provides a formal, auditable workflow for granting access for support and configuration purposes. (Note: The user interface for this feature will be released separately).

**Pre-requisites**
- Requires Super admin permissions
- No feature flags required

<Accordion title="See More">

#### Where You'll See It
- This is primarily a backend system that governs Super Admin access. The effects will be visible in audit logs and future administrative UIs.

#### What You Can Do
- Spare employees can request access to an organization for a specific duration and with a clear justification.
- Access is automatically revoked when the approved time expires.
- All requests and approvals are logged for a complete audit trail.

#### Why This Matters
This system enhances security and accountability by replacing persistent access with a "least privilege" model. It ensures that access to customer data is granted only when necessary, for a limited time, and with full transparency.

**What's New**  
The Workflows list page now includes an autocomplete search bar to quickly find workflows by name. Additionally, you can now right-click, middle-click, or Cmd/Ctrl+click on a workflow row to open it in a new tab.

**Pre-requisites**
- No feature flags required.
- Requires Admin access.

<Accordion title="See More">

#### Where You'll See It
- *Settings > System > Workflows* (in the page header).

#### What You Can Do
- Use the new search bar at the top of the page to filter the list of workflows as you type.
- Right-click any workflow row to open the browser's context menu (e.g., 'Open in new tab').
- Use standard browser shortcuts like Cmd/Ctrl+click or middle-click to open workflows in new tabs.

#### Why This Matters
Finding a specific workflow was difficult in organizations with long lists, and opening multiple workflows for comparison required manual URL handling. This change streamlines navigation and makes managing many workflows much easier and more efficient.
</Accordion>
