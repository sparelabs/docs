**What's New**  
You can now use new letter variables in Spare Engage that pull data from a rider's "Upcoming" group memberships, in addition to their "Active" ones. This is ideal for sending renewal notices or eligibility letters that need to reference future membership periods.

**Pre-requisites**
- Requires Admin access with Engage permissions

<Accordion title="See More">

#### Where You'll See It
- *Engage > Letters > [Select a Template]* (in the "Insert Variable" picker)

#### What You Can Do
- When editing a letter template, you will see new variables with an "(Including Upcoming)" suffix (e.g., `{{groupIncludingUpcoming.{groupId}.expiryTs}}`).
- These variables will use data from the most recently created membership for a given group, whether it is Active or Upcoming.
- **Example:** If a rider has an active membership expiring in Feb 2026 and an upcoming membership expiring in Feb 2027, this new variable will correctly display the 2027 expiry date.

#### Why This Matters
Previously, letter variables could only see active memberships, making it impossible to reference future eligibility periods. This change ensures you can communicate accurate, forward-looking information to riders about their upcoming memberships.
</Accordion>

### üîç Filter Cases by Connected Appointments and Letters

**What's New**  
Case list views in Spare Engage can now be filtered by their connected items, such as Appointments and Letters. This allows you to create powerful, specific views like "cases with appointments in the next 7 days" or "cases with a specific letter template sent this month."

**Pre-requisites**
- Requires Admin access with Engage permissions

<Accordion title="See More">

#### Where You'll See It
- *Engage > Eligibility > [Case Type]* (Filter dropdown)
- *Engage > Resolve > [Case Type]* (Filter dropdown)

#### What You Can Do
- In the filter dropdown on a case list, you will see new filter options with "Appt:" and "Letter:" prefixes if the case type has appointments or letters configured.
- **Appointment Filters:** Filter by Type, Status, Assigned To, Start Time, End Time, and Created At.
- **Letter Filters:** Filter by Template, Generated At, and Sent At.
- Save these filters as part of a custom view for easy access.

#### Why This Matters
Finding cases based on their related activities used to be impossible. This feature gives you the tools to quickly locate cases based on the status and timing of their associated appointments and communications, streamlining workflows for agents and managers.
</Accordion>
