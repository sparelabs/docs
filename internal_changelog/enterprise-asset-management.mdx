### üìä New Analytics Measures for Work Orders

**What's New**  
The Work Order data cube in Analytics has been enhanced with new measures to track status counts and time efficiency. This includes counts for open, pending, completed, and cancelled work orders, as well as average times to start and complete work.

**Pre-requisites**
- Requires `AnalyticsV2` feature flag
- Requires Admin access

<Accordion title="See More">

#### Where You'll See It
- *Analytics > Query Builder > Side Panel* (new measures available when the "Work Order" cube is selected)

#### What You Can Do
- **Track Status Counts**: Use the new count measures (`open_count`, `pending_count`, `completed_count`, `cancelled_count`) to get a snapshot of your work order pipeline.
- **Monitor Efficiency**: Use the new time-tracking measures to analyze performance:
    - `avg_time_to_start_hours`: Average time from when a work order is issued to when work begins.
    - `avg_time_to_complete_hours`: Average time from when a work order is issued to when it is completed.
    - `total_time_to_complete_hours`: Cumulative time spent on all completed work orders.
- **Example**: A maintenance manager can create a report with `avg_time_to_start_hours` to measure team responsiveness and identify bottlenecks in getting work started.

#### Why This Matters
These new measures provide powerful new ways to analyze maintenance operations. You can now easily quantify your team's workload, track efficiency over time, and make data-driven decisions to improve your maintenance processes.
</Accordion>
