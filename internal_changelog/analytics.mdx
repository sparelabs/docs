### üí° Clearer Tooltips for Date Filter Overrides

**What's New**  
Tooltips on analytics reports now provide clearer information about how the date filter is being applied. The icon has been updated to a filter, and the messages specify if a report is ignoring the date filter or using a custom range.

**Pre-requisites**
- Requires `AnalyticsV2` feature flag

<Accordion title="See More">

#### Where You'll See It
- *Analytics > Report View* and *Analytics > Dashboard View* (on the tooltip icon next to a report's title)

#### What You Can Do
- Hover over the new filter icon on a report to see its date filtering behavior. The tooltip will now explicitly state:
    - "Date filter is being ignored (showing all-time data)" if the override is active.
    - "Date filter is using a custom range: [start] to [end]" if a custom range is set for that report.

#### Why This Matters
Previously, it was difficult to know if a report on a dashboard was using the dashboard's main date filter or its own override. These enhanced tooltips provide immediate clarity, preventing misinterpretation of report data.
</Accordion>

### üìÖ Per-Report Date Filter Overrides on Dashboards

**What's New**  
You can now override the main dashboard date filter for individual reports on a dashboard. A new "Edit date filter" option allows you to set a custom date range or instruct a specific report to ignore the dashboard's filter entirely.

**Pre-requisites**
- Requires the `AnalyticsV2` feature flag.
- Requires Admin access.

<Accordion title="See More">

#### Where You'll See It
- *Analytics > Dashboards > [Select a Dashboard]*. In **Edit mode**, hover over a report widget and click the three-dot menu to find the new **Edit date filter** option.

#### What You Can Do
- Set a specific report to use its own date range (e.g., "Last 90 Days") while other reports on the dashboard use a different range (e.g., "Today").
- Configure a report to completely ignore the dashboard's date filter, making it static.
- Mix and match reports with different timeframes on a single dashboard.

#### Why This Matters
Sometimes, a dashboard needs to show a mix of timeframes, like "Today's Performance" alongside a "Last 30 Days Trend" report. Previously, all reports were locked to one dashboard-level filter. This change provides the flexibility to create more comprehensive and context-rich dashboards.
</Accordion>

### üìä Stacked Bar Charts Re-enabled

**What's New**  
Stacked bar charts are now available again in Analytics. When you add a second dimension to a bar chart, the bars will be stacked by default, making it easier to compare sub-groups within a larger category.

**Pre-requisites**
- Requires `AnalyticsV2` feature flag
- Requires Admin access

<Accordion title="See More">

#### Where You'll See It
- *Analytics > Query Builder* (when selecting "Bar Chart" as the chart type and adding a second dimension to your query).

#### What You Can Do
- Create reports with stacked bar charts to visualize parts of a whole. For example, you can now build a chart that shows the total number of requests per day, with each bar segmented by `Booking Channel` to see the breakdown of web, app, and phone bookings.

#### Why This Matters
Previously, this useful visualization was temporarily disabled. Re-enabling it provides a key tool for comparing the composition of different categories in your data, all within a single, easy-to-read chart.
</Accordion>

### ‚è±Ô∏è Improved Date and Time Formatting in Tables

**What's New**  
Date and time values in Analytics tables are now displayed in a cleaner, more human-readable format. Instead of showing full timestamps with milliseconds, the format now intelligently adapts to the granularity of the data (e.g., `YYYY-MM-DD` for day, `YYYY-MM-DD HH:mm` for minute).

**Pre-requisites**
- Requires `AnalyticsV2` feature flag
- Requires Admin access

<Accordion title="See More">

#### Where You'll See It
- *Analytics > Report View* (in any report displayed as a Table chart).

#### What You Can Do
- View dates and times in a consistent, easy-to-read format that omits unnecessary detail like seconds and milliseconds. For example, a timestamp that previously appeared as `2025-11-18T14:30:15.123Z` will now simply show as `2025-11-18 14:30`.

#### Why This Matters
Previously, timestamps in tables were cluttered with excessive detail, making them hard to read. This change makes tables much easier to scan and understand, improving the overall readability of your reports.
</Accordion>

- Requires Admin access

<Accordion title="See More">

#### What You Can Do
- The system now pre-selects a more logical time dimension, such as `Pickup (Time)` for trip-related reports, providing a better starting point for your analysis. You can still change it to any other time dimension.

#### Why This Matters
The previous default (`created_at`) was often not the most useful time dimension for analysis. This change provides a more intelligent default, reducing the need for manual changes and aligning the initial report more closely with common analytical questions.
</Accordion>

### üìä Download Complete Reports with No Row Limits

**What's New**  
The 10,000-row limit on downloading reports from Analytics V2 has been removed. You can now download the complete dataset for any report, regardless of size, directly from the Query Builder or the Reports List.

**Pre-requisites**
- Requires `AnalyticsV2` feature flag to access Analytics features.
- Requires Admin access.

<Accordion title="See More">

#### Where You'll See It
- *Analytics > Query Builder* (the 'Download' button in the top right).
- *Analytics > Reports* (the download action in the list of available reports).

#### What You Can Do
- Download reports in CSV or XLSX format that contain more than 10,000 rows.
- The download process now fetches data in batches, and a notification will show the progress.

#### Why This Matters
Previously, large reports were truncated at 10,000 rows, forcing users to apply narrow filters or find workarounds for complete data analysis. Now, you can export the full dataset for any query, enabling comprehensive analysis and reporting outside of Spare.
</Accordion>
