### üìû SIP Trunking Support for AI Voice

**What's New**  
AI Voice now supports receiving inbound calls via a direct SIP (Session Initiation Protocol) trunk connection. This provides a flexible alternative to using Twilio Programmable Voice for connecting your phone system to Spare's AI agents.

**Pre-requisites**
- Requires `LaunchAIVoice` feature flag

<Accordion title="See More">

#### Where You'll See It
- This is a backend capability that is configured in your telephony provider (e.g., Twilio) and affects how AI Voice agents are set up. There is no direct UI in Spare Admin for this.

#### What You Can Do
- Configure your phone numbers to use a SIP trunk to route calls directly to AI Voice.
- The system automatically detects SIP-based calls to handle functions like call transfers and recording retrieval correctly without requiring different agent configurations.

#### Why This Matters
This provides greater flexibility for integrating AI Voice with existing enterprise telephony systems. It can help simplify call routing architecture and potentially reduce telecommunications costs by allowing the use of existing SIP infrastructure.
</Accordion>

### üìû Schedule-Specific Call Forwarding Numbers

**What's New**  
You can now set a unique call forwarding phone number for each "Forwarding Hours" time rule. If a specific number isn't set for a rule, calls will be routed to the agent's default forwarding number.

**Pre-requisites**
- Requires `LaunchAIVoice` feature flag and Admin access.

<Accordion title="See More">

#### Where You'll See It
- *Admin > AI Voice > Agents > (Select an agent) > Forwarding Hours*

#### What You Can Do
- When creating or editing a "Regular Hours" or "Special Hours" rule, a new optional "Forwarding Number" field is available.
- You can enter a specific phone number for that time period. If left blank, the system uses the main forwarding number configured for the agent.
- The list of rules now includes a "Forwarding Number" column to show which number is assigned to each rule, or "Same as agent" if none is specified.

#### Why This Matters
This provides greater flexibility for call routing. For example, you can now forward calls to a different dispatch center during weekends, to an after-hours answering service overnight, or to a specific manager during holidays, all from within the same agent configuration.
</Accordion>

### üìû Granular Call Forwarding by Time of Day

**What's New**  
AI Voice now supports assigning specific forwarding phone numbers to different time rules within an agent's Forwarding Hours. This allows calls to be routed to different teams or locations based on the time of day, day of the week, or for specific date overrides like holidays.

**Pre-requisites**
- Requires `LaunchAIVoice` feature flag
- Requires Admin access

<Accordion title="See More">

#### Where You'll See It
- *Admin > AI & Messaging > Voice Agents > [Select Agent] > Forwarding Hours tab* (a new "Forwarding Number" field is now available for each time rule and override)

#### What You Can Do
- Enter a unique phone number for each scheduled time rule (e.g., weekday business hours) and for each override (e.g., holidays).
- If a time rule is active but does not have a specific number assigned, the system will fall back to using the main forwarding number set for the agent.
- If no number is set for the active rule or the agent, the call will not be forwarded.

#### Why This Matters
Previously, an AI Voice agent could only forward to a single, static phone number. This change provides the flexibility to route rider calls to the correct destination‚Äîsuch as an after-hours call center, a specific department during business hours, or a holiday support line‚Äîautomatically based on your defined schedules.
</Accordion>

### üö´ Exclude Specific Riders from AI Voice

**What's New**  
You can now create and manage an exclusion list of riders who will bypass AI Voice and be forwarded directly to a human agent when they call in. This is ideal for riders who require special assistance or have been identified as needing to speak with dispatch immediately.

**Pre-requisites**
- Requires Admin access with permissions to manage AI Voice Exclusions (`ManageAiVoiceExclusions`)

<Accordion title="See More">

#### Where You'll See It
- A new configuration page for managing AI Voice exclusions is available in your communication settings.

#### What You Can Do
- Add riders to an exclusion list by their Rider ID.
- When a rider on this list calls a phone number associated with an AI Voice agent, the system recognizes them and immediately forwards the call to the configured transfer number, skipping the AI conversation entirely.

#### Why This Matters
While AI Voice streamlines many interactions, some riders may always need to speak to a person. This feature provides the flexibility to define those exceptions, ensuring a better experience for specific riders while maintaining the efficiency of AI Voice for the majority of calls.
</Accordion>

### üë§ Anonymous Access for AI Agents

**What's New**  
A new "None" authentication type is available for AI Voice and AI Chat agents. This allows conversations to be initiated without requiring the system to identify the user first, enabling support for unauthenticated callers or chat users.

**Pre-requisites**
- Requires Admin access
- No feature flags required

<Accordion title="See More">

#### Where You'll See It
- *AI > Agents > New/Edit Agent* (new "None" option in the "Authentication" dropdown)

#### What You Can Do
- Configure an agent to not require user authentication by selecting the "None" authentication type.
- The system will automatically prevent you from assigning capabilities that require an authenticated user (e.g., "Book a trip") to an agent with "None" authentication.

#### Why This Matters
Previously, all AI agents required the user to be identified through their phone number or another method. This new option provides flexibility for use cases where the user is unknown or does not have a profile, such as providing general information to the public.
</Accordion>
